1.1 Estructuras Lineales

¿Cuándo usar cada una?

Opciones y Usos

  • Vector (ArrayList): Acceso aleatorio muy rápido. Útil cuando los datos cambian poco de tamaño o solo crecen al final.
  • Lista Enlazada (LinkedList): Inserciones y eliminaciones frecuentes en cualquier posición (especialmente al inicio).
  • Pila (Stack): Útil para historiales, deshacer/rehacer, o evaluación de expresiones. Se rige por LIFO (Último en entrar, primero en salir).
  • Cola (Queue): Procesamiento por turnos, buffers, o algoritmos como BFS. Se rige por FIFO (Primero en entrar, primero en salir).
  • Cola de Prioridad: Planificadores de tareas, algoritmo de Dijkstra, o encontrar el elemento máximo/mínimo dinámicamente.

Comparativa

Estructura Búsqueda Inserción Eliminación
Vector O(n) O(n) O(n)
Lista Enlazada O(n) O(1)* O(1)*
Pila / Cola O(n) O(1) O(1)

* Conociendo la referencia al nodo exacto que se va a modificar.

1. Lista Simple

Nodos apuntan al siguiente. Un solo sentido. Ideal para implementaciones donde el acceso principal es desde la cabeza.

2. Lista Circular Simple

El último nodo apunta de nuevo a la cabeza. Muy útil para algoritmos de planificación o buffers.

3. Lista Doblemente Ligada

Nodos apuntan al siguiente y al anterior. Permite iterar y borrar eficientemente en ambas direcciones.

4. Lista Doble Circular

Máxima flexibilidad. La cabeza y la cola están conectadas en ambas direcciones.

Linked List Cycle (Fácil) - LC 141
Dada la cabeza de una lista enlazada, determina si tiene un ciclo en su interior.
  • Pista (Floyd's Algorithm): Usa el concepto de "Liebre y Tortuga" (dos punteros, uno avanza rápido saltando de a 2, y otro lento de a 1). Si alguna vez se encuentran, hay un ciclo. (Tiempo: O(n), Memoria: O(1)).

Problema Práctico

Misión: Recorrer un árbol en Preorden e insertar el resultado en una Cola de Prioridad para ordenarlos. 

void preOrdenHaciaPQ(Nodo nodo, PriorityQueue<Integer> pq) {
    if (nodo == null) return;
    
    // 1. Procesar raíz (insertar en PQ)
    pq.offer(nodo.valor); 
    
    // 2. Izquierda
    preOrdenHaciaPQ(nodo.izq, pq);
    
    // 3. Derecha
    preOrdenHaciaPQ(nodo.der, pq);
}

Reto LeetCode (Árboles)

Lowest Common Ancestor - LCA (Medio) - LC 236
Dado un BST y dos nodos 'p' y 'q', encuentra su ancestro común más bajo (el nodo más profundo que tiene a ambos como descendientes).
  • Pista: Aprovecha la propiedad del BST. Comienza en la raíz: si 'p' y 'q' son menores, el LCA está a la izquierda. Si son mayores, está a la derecha. Si uno es menor y otro mayor (o uno es igual a la raíz), ¡entonces la raíz actual es el LCA! (Tiempo: O(log n)).

1.3 Selección de Estructuras

El mundo real

Problema 1: Búsqueda Constante

Escenario: Una empresa necesita buscar usuarios constantemente por su correo electrónico o nombre de usuario.

¿Qué usaría? (BST vs Hash vs Lista)
Estructura Búsqueda Inserción Eliminación
Lista O(n) O(1)* O(1)*
BST Equilibrado O(log n) O(log n) O(log n)
Hash Table (Promedio) O(1) O(1) O(1)

* ¡El Hash Table es la mejor opción! Las operaciones toman O(1) promedio, ideal para búsquedas exactas e instantáneas.

Problema 2: Alta Velocidad

Escenario: Se reciben millones de registros por segundo (ej. telemetría, eventos de servidor).

¿Qué estructura usaría?
  • Solución: Una Cola (Queue) o un Buffer Circular (Ring Buffer).
  • ¿Por qué? Las colas permiten una inserción asíncrona y eficiente en O(1) puro. Evita bloqueos y permite que otro hilo (consumidor) procese los registros a su ritmo siguiendo un patrón FIFO.

Retos LeetCode (Hash / Diseño)

Two Sum (Fácil) - LC 1
Dado un array de enteros y un 'target', devuelve los índices de los 2 números que suman el target.
  • Pista: Usa un HashMap para guardar el número visto y su índice. Por cada número 'x', busca si 'target - x' ya existe en el Hash. Solución en una sola pasada O(n).
LRU Cache (Medio) - LC 146
Diseña una estructura de caché que descarte el elemento menos usado recientemente (Least Recently Used) al alcanzar su capacidad máxima.
  • Pista de Diseño: Combina un HashMap (para acceso O(1)) junto con una Lista Doblemente Ligada (para mover súper rápido el elemento más usado a la cabeza, o eliminar la cola en O(1)).

1.4 Complejidad

Big O en código

Ejemplo 1


for(int i = 1; i < n; i *= 2) {
    for(int j = 0; j < n; j++) {
        // Operación de tiempo constante
    }
}
Complejidad: O(n log n)

¿Por qué?
1. El bucle interno siempre itera 'n' veces: O(n).
2. El bucle externo multiplica la variable 'i' por 2 en cada paso. Alcanzar 'n' mediante multiplicaciones sucesivas toma el logaritmo base 2 de 'n' pasos: O(log n).
3. Ambos están anidados, por lo que multiplicamos: n * log n.

Ejemplo 2


int i = 1;
while(i < n) {
    i *= 2;
}
Complejidad: O(log n)

¿Por qué?
Al igual que en el bucle externo del ejemplo anterior, la variable 'i' crece exponencialmente (1, 2, 4, 8, 16...). El número de pasos requeridos para que 'i' alcance o supere 'n' es log₂(n). No depende linealmente de n.

Ejemplo 3: O(N²)


for(int i = 0; i < n; i++) {
    for(int j = 0; j < n; j++) {
        // Operación de tiempo constante
    }
}
Complejidad: O(N²)

¿Por qué?
Para cada elemento de `n`, iteramos nuevamente `n` veces. Este es el comportamiento típico de algoritmos ingenuos como Bubble Sort.

Ejemplo 4: O(2^N)


int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
Complejidad: O(2^N)

¿Por qué?
Por cada llamada recursiva, hacemos dos llamadas más. Esto forma un árbol binario de llamadas de altura `n`, lo que resulta en un tiempo exponencial (el peor de los casos).

Desafíos de Entrevista

1. Búsqueda Binaria
En un arreglo ordenado, cortas la búsqueda a la mitad en cada paso. ¿Cuál es su complejidad?
R/ O(log N). Siempre que el tamaño del problema se divida por una fracción constante, es logarítmico.
2. Bucles Secuenciales
Un bucle itera el array A (tamaño N) y luego otro bucle separado itera el array B (tamaño M). ¿Complejidad?
R/ O(N + M). ¡NO es O(N*M)! Al no estar anidados, sus tiempos simplemente se suman.

1.5 Integración Total

Árbol → Recorrido → Cola de Prioridad → HashMap

El Reto Integral

Contamos con un Árbol Binario que almacena tareas (ID, Prioridad).

1. Recorrido: Explorar el árbol (ej. Postorden) para extraer las tareas.
2. Cola de Prioridad: Insertarlas en una Max-PriorityQueue para que las tareas más urgentes queden al frente.
3. HashMap: A medida que sacamos tareas de la cola para procesarlas, registrar su estado como 'COMPLETADA' en un HashMap utilizando su ID.

Reto LeetCode (Integración)

Merge k Sorted Lists (Difícil) - LC 23
Dadas k listas enlazadas que ya están ordenadas independientemente, únelas todas en una sola lista enlazada final, también ordenada.
  • Pista Magistral: Usa una Cola de Prioridad (Min-Heap). Inserta la cabeza de cada una de las k listas en el Heap. Extrae el menor (se unirá a tu lista de respuesta) y, si ese nodo extraído tiene un siguiente nodo en su propia lista original, insértalo de vuelta en el Heap.
  • Complejidad: Tiempo O(N log k), donde N es el total de nodos en todas las listas.
